# Big Data Concepts and PySpark Processing

## Project Overview

This project demonstrates fundamental Big Data concepts and practical data processing using PySpark. The project utilizes a sales dataset to perform distributed analytics, compare storage formats, and design a Medallion Architecture for scalable data engineering workflows.

The implementation is performed using Google Colab and Apache Spark (PySpark).

---

## Objectives

* Understand the 5 Vs of Big Data using a Flipkart case study
* Differentiate PySpark Transformations and Actions
* Perform distributed data processing using PySpark DataFrame API
* Identify the Top 5 Cities by Revenue
* Compare CSV and Parquet storage formats
* Design a Medallion Architecture (Bronze, Silver, Gold)

## Big Data Concepts

### The 5 Vs of Big Data

#### Volume

Large amounts of transaction data generated by e-commerce platforms.

#### Velocity

Continuous generation of customer orders and website interactions.

#### Variety

Structured and unstructured data such as transactions, reviews, and images.

#### Veracity

Data quality challenges such as missing values and duplicates.

#### Value

Business insights derived from analytics and reporting.

## PySpark Transformations

Transformations are lazy operations that create a new DataFrame without executing immediately.

Examples:

* select()
* filter()
* groupBy()
* withColumn()

---

## PySpark Actions

Actions trigger execution and return results.

Examples:

* show()
* count()
* collect()
* first()

## CSV vs Parquet Comparison

### CSV

Advantages:

* Human readable
* Easy to edit

Disadvantages:

* Larger file size
* Slower query performance

### Parquet

Advantages:

* Columnar storage
* Compression support
* Faster analytical queries

Disadvantages:

* Not human readable

Conclusion:

Parquet provides better storage efficiency and query performance for Big Data workloads.
